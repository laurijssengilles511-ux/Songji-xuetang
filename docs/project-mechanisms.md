# 诵记学堂项目机制梳理

本文档用于解释项目的用户机制、数据流和后续整改边界。它是开发维护文档，不面向最终用户展示。

## 一句话架构

诵记学堂是一个纯前端本地学习应用：页面直接加载 HTML、CSS、JS 和内置数据文件，账号、牌组、复习进度、统计全部保存在当前浏览器的 `localStorage` 中。

## 页面职责

- `index.html`：首页，展示今日待复习、已复习、连续天数、掌握率，并提供主要入口。
- `dictation.html`：主学习台，负责牌组选择、单元选择、SRS 复习、旧默写/Anki 练习和学习中卡片管理。
- `create.html`：自建牌组编辑器，负责单张添加、批量导入、Cloze、图片卡片、草稿恢复、编辑已有牌组。
- `profile.html`：个人主页，展示本地用户资料、自建牌组和收藏牌组。
- `browser.html`：卡片浏览器，负责搜索、筛选、暂停、搁置、重置、标记、标签、导入和导出。
- `stats.html`：统计看板，负责到期预测、状态分布、今日表现。
- `lunyu.html`：论语独立学习页，同时把评分同步到 SRS。

## 用户身份机制

项目里的“账号”是本地资料空间，不是联网账号：

- 用户数据保存在 `songji-auth-v1`。
- 登录状态只影响当前浏览器。
- 密码只经过简单编码，不能当成安全认证。
- 换浏览器、换设备、清除站点数据都会导致资料不可见，除非提前导出备份。
- 访客使用固定的 `guest` 资料空间，可以学习官方牌组。

因此界面文案应避免暗示云同步、安全登录或服务器账号。

## 内容来源

官方牌组通过全局数据对象注入：

- `window.COLLEGE_ENGLISH_IV`
- `window.RUSSIAN_WORDBOOK`
- `window.LUNYU_TRANSLATIONS`

自建牌组保存在本地用户对象的 `customBooks` 中。公开牌组对同一浏览器内所有本地用户可见，私密牌组只对创建者可见。

## SRS 卡片 ID

所有内容进入 SRS 前都会映射为统一卡片 ID：

- 英语：`english:college-english-iv:<word>`
- 俄语：`russian:russian-i:<word>`
- 论语：`lunyu:default:<sentenceId>`
- 自建：`custom:<ownerId>_<bookId>:<word>`

注意：自建卡片目前使用背面答案 `word` 作为 ID 的一部分。若同一牌组出现重复背面，或编辑背面答案，需要特别关注 ID 冲突和旧进度迁移。

## SRS 状态机制

SRS 状态保存在 `songji-srs-v1:<accountId>`，统计保存在 `songji-srs-v1:<accountId>:stats`。

核心状态：

- `new`：新卡，尚未开始。
- `learning`：学习步阶中。
- `review`：已毕业，按天复习。
- `relearning`：旧数据兼容或遗忘重学状态。

评分含义：

- `again`：再看，只在当前会话重复，不推进熟悉度。
- `hard`：困难，短间隔或较小增长。
- `good`：良好，正常推进。
- `easy`：简单，更长间隔并提高易度。

今日队列由两部分组成：

1. 已到期的 `learning`、`relearning`、`review` 卡片。
2. 当前可学习但还没有状态的新卡。

当前产品逻辑不限制主动学习新卡数。

## 旧练习数据

项目中仍保留旧默写/Anki 机制的数据：

- `songji-foreign-dictation-v1:<accountId>`
- 收藏单词、错题集、旧进度、收藏牌组等。

SRS 引擎提供迁移函数，把旧英语/论语数据转换为 SRS 状态。后续整改方向是让新功能以 SRS 为主，旧练习数据仅承担收藏和兼容职责。

## 备份与恢复

卡片浏览器提供 JSON 导出/导入：

- 导出包含本地账号、当前账号 SRS 数据和统计。
- 导入会写回本地账号和当前账号 SRS 数据。
- 这不是云同步，也不是多设备合并；更接近手动备份恢复。

## 整改优先级

1. 文案层：明确“本地资料空间”，降低用户对云账号和跨设备同步的误解。
2. 目录层：把各页面重复拼装的 SRS 牌组源和卡片 ID 逻辑收敛到公共函数。
3. 数据层：为自建卡片引入稳定 cardId，减少背面答案变更导致的状态迁移风险。
4. 代码层：把 `app.js` 拆成身份、牌组目录、SRS 内容映射、页面初始化等模块。
5. 产品层：把旧默写/Anki 与新 SRS 的边界重新设计，避免两套进度概念并存。
